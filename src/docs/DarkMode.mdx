# Dark Mode

Set `data-theme="dark"` on `<html>` (or any ancestor) to switch every component to dark mode. Each component applies its own `dark:` classes explicitly per property, rather than one global token inversion - `theme.css` registers what `dark:` means via `@custom-variant dark (&:where([data-theme="dark"], [data-theme="dark"] *));`, so the prefix means "under `data-theme="dark"`" instead of Tailwind's default `prefers-color-scheme` behavior.

```html
<html data-theme="dark">
```

This is opt-in only - deliberately **not** tied to `prefers-color-scheme`. A component library shouldn't guess your app's intended theme from OS preference, since your app may have its own theme system that should take priority; toggle the attribute yourself (e.g. from a theme switcher, or by reading `prefers-color-scheme` in your own app code if you want that behavior).

## Bringing your own theme

If you're bringing your own theme (see **Styling**), `theme.css`'s `@custom-variant dark` line isn't imported either - add that same line alongside your own `@theme` block:

```css
@custom-variant dark (&:where([data-theme="dark"], [data-theme="dark"] *));

@theme {
  --color-default-50: /* your lightest shade */;
  /* ...through 950 */
  --color-default-950: /* your darkest shade */;
}
```

You only need **one** palette (not a separate inverted dark set) - every component's stylesheet already declares which literal shade it uses for light vs. dark per property.

## Previewing it here

Use the theme toggle in this site's toolbar (top of the page) to switch between light and dark and see how each component responds.
